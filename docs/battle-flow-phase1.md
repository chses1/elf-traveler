# 第一階段對戰流程設計

## 目標

先完成一場可以從開始打到結束的最小對戰，讓玩家感受到「答題會直接影響戰鬥」。

## 流程

1. 玩家從地區介紹頁開始挑戰。
2. 系統建立戰鬥狀態：
   - 玩家 HP 100，能量 0 / 5。
   - 敵方 HP 使用該區守護精靈的最大 HP。
   - 題目池使用該地區題目。
3. 每回合顯示一題四選一。
4. 玩家答對：
   - 造成普通傷害。
   - 獲得能量。
   - 連續答對達指定門檻時獲得額外能量。
5. 玩家答錯：
   - 不獲得能量。
   - 敵方反擊，玩家扣 HP。
6. 能量足夠時可以使用大招。
7. 敵方 HP 歸零時勝利，玩家 HP 歸零時失敗。

## 第一階段暫不處理

- 戰勝後寫入收服進度。
- 戰鬥中使用道具。
- 每隻精靈的小技能差異。
- 屬性相剋。
- 敵方主動施放技能。

## 後續銜接

第二階段可以從這個基礎擴充：

- 戰後獎勵與 localStorage 儲存。
- 圖鑑依收服狀態顯示。
- 商店道具可在戰鬥中使用。
- 不同技能類型帶來治療、強化、弱化效果。
